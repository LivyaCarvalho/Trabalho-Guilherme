---

## 📌 Descrição
Este projeto foi desenvolvido com o objetivo de aplicar, na prática, conceitos de **Qualidade de Software**, por meio da implementação de testes automatizados, integração contínua (CI/CD) e revisão de código.

A aplicação consiste em uma solução web simples, com lógica em JavaScript, utilizada para simular funcionalidades básicas (como manipulação de dados ou carrinho).

---

## 🎯 Objetivo da Atividade
Aplicar conceitos de qualidade de software por meio de:

- Implementação de testes automatizados
- Uso de integração contínua (CI/CD)
- Revisão de código com Pull Request
- Uso de versionamento com GitHub

## 🧪 Testes Automatizados
Foram implementados testes automatizados para validar o funcionamento das funções principais da aplicação.

Os testes verificam regras de negócio e garantem que o sistema continue funcionando corretamente após alterações no código.

Para executar os testes localmente:

```bash
npm test

## ⚙️ Integração Contínua (CI/CD)

Foi configurado um pipeline de Integração Contínua utilizando GitHub Actions.

Sempre que ocorre:

Um push na branch main
Ou um Pull Request

O sistema executa automaticamente:

Instalação das dependências (npm install)
Execução dos testes (npm test)

O status da execução pode ser acompanhado pelo badge no topo deste documento.

##🔄 Revisão de Código (Pull Request)

A atividade incluiu a prática de revisão de código em dupla.

O fluxo adotado foi:

Um integrante realizou alterações no código
Criou um Pull Request
O outro integrante revisou e comentou
Após aprovação, o código foi integrado ao projeto

Essa prática contribui para a melhoria da qualidade do software e redução de erros.
